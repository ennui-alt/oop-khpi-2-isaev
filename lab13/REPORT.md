\# Звіт з лабораторної роботи №13



\## Тема



Обробка тексту і регулярні вирази в Java.



\## Мета роботи



Метою лабораторної роботи є набуття практичних навичок обробки текстових даних у Java, використання базових операцій класу `String`, побудова підсумкового текстового результату за допомогою `StringBuilder`, а також застосування регулярних виразів для перевірки формату, вилучення полів і перетворення текстових записів на об’єкти предметної області.



Додатково в роботі продемонстровано базову багатопотоковість у Java за допомогою `ExecutorService`.



\## Визначення варіанта



Номер у списку:



```text

N = 3

```



Формула визначення індивідуального варіанта:



```text

V = ((N - 1) mod 15) + 1

```



Підставимо значення:



```text

V = ((3 - 1) mod 15) + 1

V = (2 mod 15) + 1

V = 2 + 1

V = 3

```



Отже, індивідуальний варіант — \*\*№3 “Розклад занять”\*\*.



\## Постановка завдання



Необхідно розробити консольний Java-застосунок для обробки текстових даних відповідно до предметної області варіанта.



Для варіанта №3 потрібно обробляти записи розкладу занять. Кожен запис містить:



\- день тижня;

\- номер пари;

\- час проведення;

\- аудиторію;

\- викладача;

\- назву дисципліни.



Вхідні дані повинні зберігатися у текстовому файлі та зчитуватися програмою під час виконання. Програма не повинна обмежуватися рядками, жорстко заданими в коді демонстраційного класу.



Програма має виконувати повний цикл текстової обробки:



1\. відкриття текстового файлу;

2\. зчитування рядків;

3\. очищення та нормалізацію тексту;

4\. перевірку формату;

5\. вилучення полів із рядка;

6\. перетворення текстових даних на об’єкти;

7\. формування підсумкового звіту;

8\. обробку помилкових записів.

\## Вхідні дані



Вхідний файл розміщено за шляхом:



```text

src/main/resources/schedule.txt

```



Формат одного коректного запису:



```text

День тижня;Номер пари;Час;Аудиторія;Викладач;Предмет

```



Приклад коректного запису:



```text

Понеділок;1;08:30-10:05;305-А;Вакарчук С. І.;Об'єктно-орієнтоване програмування

```



У файлі передбачено як коректні, так і некоректні записи. Це дозволяє перевірити не тільки успішне перетворення тексту на об’єкти, а й обробку помилкових рядків.



Приклад некоректного запису з неправильним форматом:



```text

Неправильний рядок без потрібного формату

```



Приклад некоректного запису з неправильним часом:



```text

Середа;2;10:25-09:00;301;Кароль Т. Г.;Некоректний час

```



У другому випадку час початку пари пізніший за час завершення, тому такий запис вважається помилковим.



\## Опис класів програми



\## Клас `Main`



Клас `Main` є демонстраційним класом програми.



Він виконує такі дії:



\- зчитує файл `schedule.txt` з каталогу ресурсів;

\- створює об’єкт класу `ScheduleProcessor`;

\- запускає обробку рядків;

\- отримує результат обробки;

\- виводить сформований звіт у консоль.



Основний запуск програми відбувається в методі:



```java

public static void main(String\[] args)

```



Для зчитування файлу використовується допоміжний метод:



```java

private static List<String> readFileFromResources(String fileName)

```



Цей метод шукає файл у ресурсах проєкту та зчитує всі його рядки за допомогою `Files.readAllLines()`.



\## Клас `Lesson`



Клас `Lesson` є класом предметної області.



Він описує одне заняття з розкладу та містить такі поля:



\- `day` — день тижня;

\- `number` — номер пари;

\- `startTime` — час початку;

\- `endTime` — час завершення;

\- `classroom` — аудиторія;

\- `teacher` — викладач;

\- `subject` — назва дисципліни.



Клас також має метод:



```java

public String getTime()

```



Він повертає час заняття у форматі:



```text

08:30-10:05

```



Метод `toString()` використовується для зручного формування текстового подання заняття у звіті.



\## Перелік `WeekDay`



`WeekDay` — це перелік днів тижня.



Він містить такі значення:



```text

MONDAY

TUESDAY

WEDNESDAY

THURSDAY

FRIDAY

SATURDAY

SUNDAY

```



Кожен день має текстову назву українською мовою та порядковий номер. Порядковий номер використовується для сортування занять у правильному порядку.



Метод:



```java

public static WeekDay fromText(String text)

```



перетворює текстовий день тижня з файлу на відповідне значення enum.



Наприклад:



```text

Понеділок -> MONDAY

Вівторок -> TUESDAY

```



\## Клас `ScheduleResult`



Клас `ScheduleResult` зберігає результат обробки файлу.



Він містить два списки:



```java

private final List<Lesson> lessons = new ArrayList<>();

private final List<String> errors = new ArrayList<>();

```



Список `lessons` містить коректно оброблені заняття.



Список `errors` містить повідомлення про помилки для некоректних рядків.



Клас має методи:



```java

public void addLesson(Lesson lesson)

public void addError(String error)

public List<Lesson> getLessons()

public List<String> getErrors()

```



Таким чином, програма не завершується при першій помилці, а продовжує обробку інших рядків.



\## Клас `ScheduleProcessor`



`ScheduleProcessor` є основним сервісним класом програми.



Він відповідає за всю логіку текстової обробки:



\- нормалізацію рядків;

\- перевірку формату;

\- вилучення полів;

\- створення об’єктів `Lesson`;

\- багатопотокову обробку;

\- сортування розкладу;

\- формування підсумкового звіту.



\## Нормалізація тексту



Для попередньої обробки рядків використовується метод:



```java

public String normalizeLine(String line)

```



Він виконує такі дії:



\- перевіряє, чи рядок не є `null`;

\- прибирає зайві пробіли на початку і в кінці;

\- замінює декілька пробілів одним;

\- прибирає зайві пробіли біля символу `;`.



Приклад:



```text

&#x20; Понеділок ; 1 ; 08:30-10:05 ; 305-А ; Вакарчук С. І. ; Java

```



після нормалізації перетворюється на:



```text

Понеділок;1;08:30-10:05;305-А;Вакарчук С. І.;Java

```



У цьому методі використовуються базові операції класу `String`:



\- `trim()`;

\- `replaceAll()`;

\- `isBlank()`.



\## Регулярний вираз



Для перевірки формату рядка використовується регулярний вираз `LINE\_PATTERN`.



Він описує структуру запису розкладу:



```java

private static final Pattern LINE\_PATTERN = Pattern.compile(

&#x20;       "^(Понеділок|Вівторок|Середа|Четвер|П\['’]ятниця|Субота|Неділя);" +

&#x20;               "(\[1-9]);" +

&#x20;               "(\\\\d{2}:\\\\d{2})-(\\\\d{2}:\\\\d{2});" +

&#x20;               "(\[A-ZА-ЯІЇЄҐ]?\\\\d{2,3}(-\[А-ЯA-ZІЇЄҐ])?);" +

&#x20;               "(\[^;]+);" +

&#x20;               "(.+)$",

&#x20;       Pattern.CASE\_INSENSITIVE | Pattern.UNICODE\_CASE

);

```



Цей регулярний вираз перевіряє:



\- коректну назву дня тижня;

\- номер пари;

\- час початку;

\- час завершення;

\- формат аудиторії;

\- наявність викладача;

\- наявність назви предмета.



У роботі використано не тільки `matches()`, а й класи `Pattern` та `Matcher`.



\## Вилучення груп із тексту



Після успішної перевірки формату дані вилучаються через групи `Matcher`.



У методі `parseLine()` використовуються такі групи:



```java

WeekDay day = WeekDay.fromText(matcher.group(1));

int number = Integer.parseInt(matcher.group(2));



LocalTime startTime = LocalTime.parse(matcher.group(3));

LocalTime endTime = LocalTime.parse(matcher.group(4));



String classroom = matcher.group(5).trim();

String teacher = matcher.group(7).trim();

String subject = matcher.group(8).trim();

```



Значення груп:



```text

group(1) — день тижня

group(2) — номер пари

group(3) — час початку

group(4) — час завершення

group(5) — аудиторія

group(7) — викладач

group(8) — предмет

```



Таким чином, програма не просто перевіряє рядок, а вилучає з нього всі потрібні поля.



\## Перетворення тексту на об’єкти



Після вилучення полів програма створює об’єкт класу `Lesson`:



```java

return new Lesson(day, number, startTime, endTime, classroom, teacher, subject);

```



Це означає, що сирий текстовий рядок перетворюється на структурований об’єкт предметної області.



Наприклад, рядок:



```text

Понеділок;1;08:30-10:05;305-А;Вакарчук С. І.;Об'єктно-орієнтоване програмування

```



перетворюється на об’єкт `Lesson`, який містить окремі значення дня, номера пари, часу, аудиторії, викладача та предмета.



\## Перевірка часу



Окремо в програмі перевіряється логічна правильність часу.



Якщо час початку не є ранішим за час завершення, генерується помилка:



```java

if (!startTime.isBefore(endTime)) {

&#x20;   throw new IllegalArgumentException("початок пари пізніше або дорівнює кінцю: " + line);

}

```



Тому запис:



```text

Середа;2;10:25-09:00;301;Кароль Т. Г.;Некоректний час

```



не додається до розкладу, а потрапляє до списку помилок.



\## Багатопотоковість



У роботі реалізована базова багатопотоковість.



Для цього використовується метод:



```java

public ScheduleResult processLinesInThreads(List<String> lines) throws InterruptedException

```



У цьому методі створюється пул із трьох потоків:



```java

ExecutorService executor = Executors.newFixedThreadPool(3);

```



Для кожного рядка файлу створюється окрема задача:



```java

Callable<LineResult> task = () -> handleOneLine(lineNumber, line);

```



Після цього задача передається на виконання:



```java

futures.add(executor.submit(task));

```



Результати виконання задач збираються через `Future`:



```java

LineResult lineResult = future.get();

```



Це дозволяє обробляти декілька рядків файлу паралельно.



У консолі додатково виводиться назва потоку:



```java

String threadName = Thread.currentThread().getName();



System.out.println("Потік " + threadName + " обробляє рядок №" + lineNumber);

```



Приклад виводу:



```text

Потік pool-1-thread-1 обробляє рядок №1

Потік pool-1-thread-2 обробляє рядок №2

Потік pool-1-thread-3 обробляє рядок №3

```



Це підтверджує, що рядки обробляються не тільки в головному потоці `main`, а в окремих потоках пулу.



\## Обробка помилок



Помилкові рядки не зупиняють виконання всієї програми.



Кожен рядок обробляється в методі:



```java

private LineResult handleOneLine(int lineNumber, String line)

```



Якщо рядок коректний, повертається успішний результат:



```java

return LineResult.success(lesson);

```



Якщо під час обробки виникає помилка, вона записується у результат:



```java

return LineResult.error("рядок №" + lineNumber + ": " + e.getMessage());

```



Завдяки цьому програма може одночасно показати:



\- скільки записів оброблено успішно;

\- які рядки містять помилки;

\- чому саме ці рядки не були прийняті.



\## Сортування розкладу



Після обробки рядків коректні заняття сортуються.



Сортування виконується за:



1\. днем тижня;

2\. номером пари;

3\. часом початку.



Для цього використовується `Comparator`:



```java

result.getLessons().sort(

&#x20;       Comparator.comparingInt((Lesson lesson) -> lesson.getDay().getOrder())

&#x20;               .thenComparingInt(Lesson::getNumber)

&#x20;               .thenComparing(Lesson::getStartTime)

);

```



Це дозволяє отримати впорядкований розклад у зручному вигляді.



\## Формування звіту



Для формування підсумкового текстового результату використовується `StringBuilder`.



Звіт формується в методі:



```java

public String buildReport(ScheduleResult result)

```



У звіті виводиться:



\- заголовок;

\- кількість коректних записів;

\- кількість помилкових записів;

\- впорядкований розклад;

\- список помилок.



Фрагмент формування звіту:



```java

StringBuilder builder = new StringBuilder();



builder.append("Звіт з обробки розкладу занять\\n");

builder.append("================================\\n");

builder.append("Коректних записів: ").append(result.getLessons().size()).append('\\n');

builder.append("Помилкових записів: ").append(result.getErrors().size()).append("\\n\\n");

```



Використання `StringBuilder` є доцільним, оскільки звіт формується поступово з багатьох рядків.

\## Тестовий клас `ScheduleProcessorTest`



Цей клас перевіряє основну логіку обробки текстових даних.



У ньому реалізовано такі тести:



\### `normalizeLineShouldRemoveExtraSpaces`



Перевіряє, що метод `normalizeLine()` правильно прибирає зайві пробіли та нормалізує рядок.



\### `parseLineShouldCreateLessonFromCorrectLine`



Перевіряє, що коректний текстовий запис правильно перетворюється на об’єкт `Lesson`.



У тесті перевіряються:



\- день тижня;

\- номер пари;

\- час;

\- аудиторія;

\- викладач;

\- предмет.



\### `parseLineShouldRejectWrongFormat`



Перевіряє, що рядок із неправильним форматом не приймається програмою.



\### `parseLineShouldRejectWrongTimeOrder`



Перевіряє, що запис із неправильним часом відхиляється.



\### `processLinesInThreadsShouldSeparateCorrectAndWrongLines`



Перевіряє, що програма правильно розділяє коректні та помилкові записи.



\### `processLinesInThreadsShouldSortLessonsByDayAndNumber`



Перевіряє правильність сортування занять за днем тижня та номером пари.



\### `buildReportShouldContainMainInformation`



Перевіряє, що сформований звіт містить основну інформацію:



\- заголовок;

\- кількість коректних записів;

\- кількість помилкових записів;

\- дані заняття;

\- блок помилок.



\## Тестовий клас `MainTest`

\## Висновок



У ході виконання лабораторної роботи було розроблено консольний Java-застосунок для обробки текстового розкладу занять. Програма зчитує дані з файлу, нормалізує рядки, перевіряє їх регулярним виразом і вилучає потрібні поля за допомогою `Pattern` та `Matcher`.



Коректні записи перетворюються на об’єкти класу `Lesson`, а помилкові рядки зберігаються у списку помилок. Для формування підсумкового багаторядкового звіту використано `StringBuilder`.



Додатково в роботі реалізовано багатопотокову обробку рядків через `ExecutorService`, `Callable` і `Future`. Це дозволило показати базовий принцип паралельного виконання задач у Java.



Також було створено модульні тести JUnit, які перевіряють нормалізацію тексту, розбір коректного запису, відхилення помилкових рядків, сортування розкладу та формування звіту. Отже, основні вимоги лабораторної роботи виконано.
