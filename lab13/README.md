\# Лабораторна робота №13



\## Тема



Обробка тексту і регулярні вирази в Java.



\## Варіант



Номер у списку: `N = 3`.



Формула визначення варіанта:



```text

V = ((N - 1) mod 15) + 1

V = ((3 - 1) mod 15) + 1 = 3

```



Отже, індивідуальний варіант — \*\*№3 “Розклад занять”\*\*.



\## Опис програми



Програма є консольним Java-застосунком для обробки розкладу занять.



Вхідні дані зберігаються у текстовому файлі `schedule.txt`. Кожен рядок файлу містить інформацію про одне заняття:



```text

День тижня;Номер пари;Час;Аудиторія;Викладач;Предмет

```



Приклад запису:



```text

Понеділок;1;08:30-10:05;305-А;Вакарчук С. І.;Об'єктно-орієнтоване програмування

```



Програма зчитує файл, нормалізує рядки, перевіряє їх регулярним виразом, вилучає потрібні поля та перетворює їх на об’єкти предметної області.



\## Основні можливості



\- зчитування даних із текстового файлу;

\- очищення та нормалізація рядків;

\- перевірка формату записів через регулярний вираз;

\- використання `Pattern` і `Matcher`;

\- вилучення дня тижня, номера пари, часу, аудиторії, викладача та предмета;

\- створення об’єктів `Lesson`;

\- обробка коректних і помилкових записів;

\- багатопотокова обробка рядків через `ExecutorService`;

\- формування підсумкового звіту через `StringBuilder`;

\- модульне тестування за допомогою `JUnit 5`.



\## Багатопотоковість



У програмі реалізована базова багатопотоковість.



Кожен рядок вхідного файлу передається як окрема задача в пул потоків. Для цього використовуються:



\- `ExecutorService`;

\- `Executors.newFixedThreadPool(3)`;

\- `Callable`;

\- `Future`.



У консолі виводиться назва потоку, який обробляє конкретний рядок, наприклад:



```text

Потік pool-1-thread-1 обробляє рядок №1

Потік pool-1-thread-2 обробляє рядок №2

Потік pool-1-thread-3 обробляє рядок №3

```

\## Основні класи



\### `Main`



Демонстраційний клас програми.



Відповідає за запуск програми, зчитування файлу `schedule.txt`, створення об’єкта `ScheduleProcessor` та виведення результату в консоль.



\### `ScheduleProcessor`



Основний клас для обробки текстових даних.



Виконує нормалізацію рядків, перевірку регулярним виразом, вилучення полів, створення об’єктів `Lesson`, багатопотокову обробку та формування звіту.



\### `Lesson`



Клас предметної області, який описує одне заняття.



Містить день тижня, номер пари, час початку, час завершення, аудиторію, викладача та назву предмета.



\### `WeekDay`



Перелік днів тижня.



Використовується для перетворення текстового дня тижня на enum та для сортування розкладу.



\### `ScheduleResult`



Клас для збереження результату обробки.



Містить список коректно оброблених занять і список помилок.



\## Вхідний файл



Файл з вхідними даними знаходиться за шляхом:



```text

src/main/resources/schedule.txt

```



У файлі є як коректні, так і некоректні записи. Це дозволяє показати успішну обробку правильних рядків і роботу з помилковими записами.



\## Приклад результату роботи



Після запуску програма виводить інформацію про роботу потоків і підсумковий звіт:



```text

Звіт з обробки розкладу занять

================================

Коректних записів: 6

Помилкових записів: 2

```



Також у результаті виводиться впорядкований розклад занять за днями тижня та список помилкових рядків.



\## Тестування



Для тестування використовується `JUnit 5`.



У проєкті є два тестові класи:



```text

MainTest

ScheduleProcessorTest

```



Тести перевіряють:



\- коректну нормалізацію тексту;

\- розбір валідного запису;

\- відхилення невалідного запису;

\- відхилення запису з неправильним часом;

\- поділ записів на коректні та помилкові;

\- сортування занять;

\- формування підсумкового звіту;

\- демонстраційний запуск програми.


\## Висновок



У роботі реалізовано консольний Java-застосунок для обробки текстового розкладу занять. Програма зчитує записи з файлу, нормалізує їх, перевіряє формат через регулярний вираз і перетворює коректні рядки на об’єкти `Lesson`. Для формування результату використано `StringBuilder`, а для демонстрації багатопотоковості — `ExecutorService`. Основна логіка програми перевіряється модульними тестами `JUnit 5`.
