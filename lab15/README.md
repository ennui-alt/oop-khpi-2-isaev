\# Лабораторна робота №15



\## Тема



Синхронізація потоків у Java.



\## Варіант



\*\*Варіант 3 — бронювання номерів.\*\*



У роботі реалізовано консольний Java-застосунок, у якому кілька потоків одночасно намагаються забронювати місця в готелі.



Спільним ресурсом є кімнати готелю, а змінюваним станом — кількість заброньованих місць.



\## Мета роботи



Мета роботи — показати проблему спільного доступу до змінюваних даних у багатопотоковій програмі та реалізувати коректний доступ за допомогою `synchronized`.



\## Основна ідея



У програмі реалізовано два варіанти бронювання:



\- `bookUnsafe()` — небезпечний варіант без синхронізації;

\- `bookSafe()` — безпечний варіант із використанням `synchronized`.



Кілька потоків запускаються за допомогою методу `start()`, а завершення їх роботи очікується через `join()`.

\## Основні класи



\- `HotelRoom` — модель кімнати готелю зі станом бронювання.

\- `BookingRequest` — заявка на бронювання.

\- `BookingService` — сервіс для обробки заявок.

\- `BookingTask` — задача, яка виконується в окремому потоці.

\- `Main` — демонстраційний запуск програми.



\## Запуск програми



```bash

mvn clean compile

mvn exec:java

```



Або запустити клас:



```text

ua.khpi.oop.lab15.demo.Main

```



\## Запуск тестів



```bash

mvn test

```



\## Результат роботи



Програма показує різницю між несинхронізованим і синхронізованим доступом до спільного ресурсу.



У безпечному варіанті кількість заброньованих місць не перевищує місткість готелю, оскільки критична секція захищена за допомогою `synchronized`.



\## Висновок



У лабораторній роботі було реалізовано багатопотокове бронювання номерів. Програма демонструє проблему стану перегонів у небезпечному варіанті та правильну роботу синхронізованого варіанта.
