\# Лабораторна робота №8



\## Тема



Файлове введення-виведення, серіалізація та винятки в Java.



\## Варіант



Варіант №3 — Інформаційні сервіси кампусу.



\## Опис роботи



У цій лабораторній роботі реалізовано Java-застосунок для роботи з інформаційними сервісами кампусу.



Програма дозволяє створювати записи про навчальні події, додавати їх до реєстру, переглядати список записів, виконувати пошук, зберігати дані у текстовий файл, читати їх з текстового файлу, а також виконувати серіалізацію та десеріалізацію у бінарний файл.



Основні класи предметної області:



\- `ExamReminder` — нагадування про іспит;

\- `RoomBooking` — бронювання аудиторії;

\- `ConsultationSlot` — слот консультації з викладачем.



Також використано базовий абстрактний клас `CampusService`, у якому зберігаються спільні поля для всіх сервісів: `id`, `studentName`, `date`, `time`.

\## Основні можливості програми



У програмі реалізовано:



\- створення об’єктів предметної області;

\- додавання записів до реєстру;

\- перегляд усіх записів;

\- користувацький виняток `InvalidCampusServiceException`.



\## Текстовий формат збереження



Кожен запис зберігається в окремому рядку. Поля розділяються символом `;`.



Загальний формат:



```text

TYPE;id;studentName;date;time;field1;field2

```



Формат для нагадування про іспит:



```text

EXAM;id;studentName;date;time;courseName;auditorium

```



Формат для бронювання аудиторії:



```text

ROOM;id;studentName;date;time;roomNumber;purpose

```



Формат для консультації:



```text

CONSULTATION;id;studentName;date;time;teacherName;subject

```



Приклади записів:



```text

EXAM;E001;Zakhar Isaiev;2026-05-20;10:30;Object-Oriented Programming;301

ROOM;R001;Zakhar Isaiev;2026-05-21;14:00;204;Team project meeting

CONSULTATION;C001;Zakhar Isaiev;2026-05-22;12:15;Valentina V.;Java serialization

```



\## Серіалізація



Для серіалізації використовується бінарний файл:



`build/lab08/campus-services.bin`



Серіалізація виконується у класі `CampusServiceFileManager` за допомогою `ObjectOutputStream`.



Десеріалізація виконується за допомогою `ObjectInputStream`.



Текстовий файл використовується для зручного перегляду даних, а бінарний файл — для швидкого збереження та відновлення всього стану реєстру.



\## Обробка винятків



У програмі використовується користувацький виняток `InvalidCampusServiceException`.



Він застосовується для обробки помилок предметної області, наприклад:



\- неправильна кількість полів у текстовому файлі;

\- невідомий тип запису;

\- дублювання `id`;

\- конфлікт записів за датою і часом.



Також у програмі можуть виникати стандартні винятки:



\- `IOException` — помилки читання або запису файлів;

\- `ClassNotFoundException` — помилка під час десеріалізації;

\- `IllegalArgumentException` — неправильні значення під час створення об’єктів.



\## Приклад помилкової ситуації



У програмі створюється файл:



`build/lab08/broken-campus-services.txt`



У ньому міститься неправильний рядок:



```text

EXAM;BAD\_LINE;Zakhar Isaiev

```



Цей рядок має тільки 3 поля замість 7, тому програма визначає помилку формату.



Після цього виняток обробляється, і програма виводить повідомлення про помилку, а не завершується аварійно.
