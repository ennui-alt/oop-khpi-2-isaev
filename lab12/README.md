\# Лабораторна робота №12



\## Тема



Stream API в Java.



\## Варіант



Варіант №3 — \*\*Online Store\*\*.



Предметна область проєкту — інтернет-магазин. У програмі реалізовано роботу з товарами, покупцями та замовленнями. Основна обробка даних виконується за допомогою Stream API.



\## Мета



Мета роботи — навчитися використовувати Stream API для обробки колекцій об’єктів у Java.



У проєкті показано:



\- фільтрацію даних;

\- перетворення об’єктів;

\- сортування;

\- групування;

\- підрахунок;

\- статистичне зведення;

\- формування текстового звіту.



\## Структура проєкту



```text

pom.xml

src/main/java/ua/khpi/oop/lab12/demo/Main.java

src/main/java/ua/khpi/oop/lab12/model/Product.java

src/main/java/ua/khpi/oop/lab12/model/Customer.java

src/main/java/ua/khpi/oop/lab12/model/Order.java

src/main/java/ua/khpi/oop/lab12/model/OrderStatus.java

src/main/java/ua/khpi/oop/lab12/service/OrderService.java

src/test/java/ua/khpi/oop/lab12/demo/MainTest.java

src/test/java/ua/khpi/oop/lab12/service/OrderServiceTest.java

```



\## Основні класи



| Клас | Призначення |

|---|---|

| `Product` | Описує товар інтернет-магазину |

| `Customer` | Описує покупця |

| `Order` | Описує замовлення |

| `OrderStatus` | Зберігає можливі статуси замовлення |

| `OrderService` | Виконує обробку замовлень через Stream API |

| `Main` | Демонструє роботу програми |



\## Реалізовані можливості



У програмі реалізовано:



\- створення колекції замовлень;

\- фільтрацію замовлень за статусом;

\- отримання списку унікальних назв товарів;

\- сортування замовлень за сумою;

\- підрахунок доходу із завершених замовлень;

\- групування замовлень за статусом;

\- статистику за сумами замовлень;

\- формування короткого текстового звіту;

\- порівняння звичайного циклу та Stream API.



\## Використані Stream API операції



У роботі використано:



\- `filter()`;

\- `map()`;

\- `flatMap()`;

\- `distinct()`;

\- `sorted()`;

\- `toList()`;

\- `sum()`;

\- `groupingBy()`;

\- `counting()`;

\- `summarizingDouble()`;

\- `joining()`.



\## Запуск програми



Скомпілювати проєкт можна командою:



```bash

mvn compile

```



Запустити програму можна через клас:



```text

ua.khpi.oop.lab12.demo.Main

```



Або безпосередньо через IntelliJ IDEA.



\## Запуск тестів



Для запуску тестів використовується команда:



```bash

mvn test

```



\## Тестування



У проєкті створено два тестові класи:



\- `OrderServiceTest`;

\- `MainTest`.



`OrderServiceTest` перевіряє логіку сервісу та результати stream-операцій.



`MainTest` перевіряє запуск демонстраційної програми та основний консольний вивід.



\## UML-діаграма



Для проєкту підготовлено UML-діаграму, яка показує зв’язки між класами `Product`, `Customer`, `Order`, `OrderStatus`, `OrderService`, `Main` і тестовими класами.



\## Висновок



У лабораторній роботі реалізовано консольний Java-застосунок для предметної області інтернет-магазину. Програма демонструє використання Stream API для фільтрації, сортування, групування, підрахунку, статистики та формування текстового звіту. Модульні тести підтверджують правильність роботи основних методів.
