Клас використовується в замовленнях для збереження інформації про покупця.



\### Клас `Order`



Клас `Order` описує замовлення.



Поля класу:



| Поле | Тип | Опис |

|---|---|---|

| `number` | `int` | Номер замовлення |

| `customer` | `Customer` | Покупець |

| `products` | `List<Product>` | Список товарів |

| `status` | `OrderStatus` | Статус замовлення |

| `date` | `LocalDate` | Дата замовлення |



У класі `Order` реалізовано метод `getTotalPrice()`, який обчислює загальну суму замовлення.



```java

public double getTotalPrice() {

&#x20;   return products.stream()

&#x20;           .mapToDouble(Product::getPrice)

&#x20;           .sum();

}

```



У цьому методі використовується Stream API: список товарів перетворюється на потік, після чого кожен товар перетворюється на його ціну, а потім виконується підрахунок суми.



\### Перелік `OrderStatus`

| Статус | Опис |

| `CANCELLED` | Скасоване замовлення |



Кожен статус має текстову назву, яка використовується під час виведення результатів.

\## Реалізація Stream API операцій



\### Фільтрація замовлень через `filter()`



Метод `findOrdersByStatus()` використовується для пошуку замовлень із певним статусом.



```java

public List<Order> findOrdersByStatus(OrderStatus status) {

&#x20;   return orders.stream()

&#x20;           .filter(order -> order.getStatus() == status)

&#x20;           .toList();

}

```



Операція `filter()` залишає тільки ті замовлення, які відповідають заданій умові.



\### Перетворення даних через `map()`



Операція `map()` використовується для перетворення об’єктів одного типу в інший.



У програмі вона використовується, наприклад, для перетворення товарів у їхні назви:



```java

.map(Product::getName)

```



\### Вкладене перетворення через `flatMap()`



Оскільки кожне замовлення містить список товарів, для отримання одного загального потоку всіх товарів використовується `flatMap()`.



```java

orders.stream()

&#x20;       .flatMap(order -> order.getProducts().stream())

```



Це дозволяє працювати не з окремими списками товарів у кожному замовленні, а з одним спільним потоком товарів.



\### Видалення повторів через `distinct()`



Після отримання назв товарів використовується `distinct()`, щоб прибрати дублікати.



```java

.distinct()

```



Це потрібно, тому що один і той самий товар може зустрічатися у кількох замовленнях.



\### Сортування через `sorted()`



Для сортування назв товарів використовується `sorted()`.



```java

.sorted()

```



Для сортування замовлень за сумою використано `Comparator`.



```java

.sorted(Comparator.comparingDouble(Order::getTotalPrice).reversed())

```



Такий запис сортує замовлення від найбільшої суми до найменшої.



\### Агрегування через `sum()`



Для підрахунку доходу із завершених замовлень використовується `sum()`.



```java

public double calculateIncomeFromCompletedOrders() {

&#x20;   return orders.stream()

&#x20;           .filter(order -> order.getStatus() == OrderStatus.COMPLETED)

&#x20;           .mapToDouble(Order::getTotalPrice)

&#x20;           .sum();

}

```



У цьому методі спочатку відбираються тільки завершені замовлення, після цього кожне замовлення перетворюється на його суму, а потім обчислюється загальна сума.



\### Групування через `groupingBy()`



Метод `groupOrdersByStatus()` групує замовлення за статусом.



```java

public Map<OrderStatus, List<Order>> groupOrdersByStatus() {

&#x20;   return orders.stream()

&#x20;           .collect(Collectors.groupingBy(Order::getStatus));

}

```



Результатом є мапа, у якій ключем є статус замовлення, а значенням — список замовлень із цим статусом.



\### Підрахунок через `counting()`



Метод `countOrdersByStatus()` підраховує кількість замовлень для кожного статусу.



```java

public Map<OrderStatus, Long> countOrdersByStatus() {

&#x20;   return orders.stream()

&#x20;           .collect(Collectors.groupingBy(

&#x20;                   Order::getStatus,

&#x20;                   Collectors.counting()

&#x20;           ));

}

```



Тут використано складніший конвеєр: `groupingBy()` групує замовлення, а `counting()` підраховує кількість елементів у кожній групі.



\### Статистика через `summarizingDouble()`



Метод `getOrderPriceStatistics()` формує статистику за сумами замовлень.



```java

public DoubleSummaryStatistics getOrderPriceStatistics() {

&#x20;   return orders.stream()

&#x20;           .collect(Collectors.summarizingDouble(Order::getTotalPrice));

}

```



Цей колектор дозволяє отримати:



\- кількість замовлень;

\- мінімальну суму;

\- максимальну суму;

\- середнє значення;

\- загальну суму.



\### Побудова текстового звіту через `joining()`



Метод `buildOrdersReport()` формує текстовий звіт по замовленнях.



```java

public String buildOrdersReport() {

&#x20;   return orders.stream()

&#x20;           .sorted(Comparator.comparing(Order::getDate))

&#x20;           .map(order -> "Order #" + order.getNumber()

&#x20;                   + " | " + order.getCustomer().getFullName()

&#x20;                   + " | " + order.getStatus().getTitle()

&#x20;                   + " | " + String.format("%.2f UAH", order.getTotalPrice()))

&#x20;           .collect(Collectors.joining("\\n"));

}

```



У цьому методі замовлення спочатку сортуються за датою, потім кожне замовлення перетворюється на рядок звіту, а після цього всі рядки об’єднуються в один текст.

\## Порівняння імперативного підходу та Stream API



У роботі реалізовано два способи фільтрації замовлень за статусом.



\### Імперативний підхід



```java

public List<Order> findOrdersByStatusImperative(OrderStatus status) {

&#x20;   List<Order> result = new ArrayList<>();



&#x20;   for (Order order : orders) {

&#x20;       if (order.getStatus() == status) {

&#x20;           result.add(order);

&#x20;       }

&#x20;   }



&#x20;   return result;

}

```



У цьому варіанті програміст явно створює новий список, проходить по колекції циклом, перевіряє умову та вручну додає елементи.



\### Декларативний підхід через Stream API



```java

public List<Order> findOrdersByStatus(OrderStatus status) {

&#x20;   return orders.stream()

&#x20;           .filter(order -> order.getStatus() == status)

&#x20;           .toList();

}

```



У цьому варіанті код напряму описує, що потрібно зробити: відфільтрувати замовлення за статусом і зібрати результат у список.



Stream API краще передає намір програми, тому що код стає коротшим і більш декларативним. Імперативний варіант теж зрозумілий, але містить більше технічних дій.



У цій роботі stream-конвеєри залишаються читабельними, тому що кожна операція винесена в окремий метод із зрозумілою назвою.
