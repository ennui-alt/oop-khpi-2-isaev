\# Лабораторна робота №11



\## Тема



Collections Framework в Java.



\## Мета роботи



Метою лабораторної роботи є формування практичних навичок використання стандартних колекцій Java Collections Framework.



У роботі потрібно навчитися обирати відповідну колекцію для конкретного завдання, зберігати об’єкти предметної області, виконувати базові операції над ними та пояснювати, чому було обрано саме такі структури даних.



\## Варіант



Номер у списку групи:



```text

N = 3

```



Формула визначення варіанта:



```text

V = ((N - 1) mod 15) + 1

```



Обчислення:



```text

V = ((3 - 1) mod 15) + 1 = 3

```



Отже, індивідуальний варіант — \*\*№3\*\*.



\## Предметна область



Предметна область лабораторної роботи — \*\*планувальник завдань команди\*\*.



У програмі реалізовано сервіс для роботи із задачами команди розробників. Сервіс дозволяє додавати задачі, переглядати список задач, знаходити задачу за ідентифікатором, змінювати статус задачі, видаляти задачу, переглядати розробників команди та працювати з чергою задач.



\## Основні класи



У проєкті реалізовано такі класи:



\- `Task` — описує задачу команди.

\- `Developer` — описує розробника команди.

\- `Sprint` — описує спринт, у межах якого виконуються задачі.

\- `TaskStatus` — містить можливі статуси задачі.

\- `TeamTaskPlanner` — сервісний клас для роботи з колекціями.

\- `Main` — демонстраційна програма.



\## Використані колекції



У класі `TeamTaskPlanner` використано три стандартні колекції Java:



```java

private final List<Task> tasks = new ArrayList<>();

private final Queue<Task> taskQueue = new LinkedList<>();

private final Set<Developer> developers = new LinkedHashSet<>();

```



\## Обґрунтування вибору колекцій



\### `List<Task>`



Колекція `List<Task>` використовується як основне внутрішнє сховище задач.



Вона підходить для цієї роботи, тому що дозволяє зберігати задачі у порядку додавання, переглядати всі задачі та виконувати послідовний пошук.



\### `Queue<Task>`



Колекція `Queue<Task>` використовується для організації черги задач на виконання.



Задачі обробляються за принципом FIFO:



```text

First In — First Out

```



Це означає, що задача, яка була додана першою, першою потрапляє на виконання.



\### `Set<Developer>`



Колекція `Set<Developer>` використовується для збереження унікальних розробників команди.



У класі `Developer` методи `equals()` та `hashCode()` реалізовані на основі поля `email`, тому один і той самий розробник не додається повторно.

\## Додаткове завдання



У роботі також виконано частину додаткового завдання.



\### Сортування через `Comparator`



Для сортування задач за пріоритетом реалізовано метод `getTasksSortedByPriority()`.



```java

public List<Task> getTasksSortedByPriority() {

&#x20;   List<Task> sortedTasks = new ArrayList<>(tasks);

&#x20;   sortedTasks.sort(Comparator.comparingInt(Task::getPriority));

&#x20;   return sortedTasks;

}

```



Метод створює копію списку задач і сортує її за пріоритетом. Основний список задач при цьому не змінюється.



\### Усунення дублікатів через `Set`



Для збереження розробників використано `Set<Developer>`. Завдяки цьому однакові розробники не додаються повторно.



\### Черга задач через `Queue`



Для організації черги задач використано `Queue<Task>`. Наступна задача отримується методом `takeNextTask()`.



```java

public Task takeNextTask() {

&#x20;   return taskQueue.poll();

}

```
