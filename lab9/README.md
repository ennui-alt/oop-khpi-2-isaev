\# Лабораторна робота №9



\## Тема



Узагальнення (Generics) в Java.



\## Варіант



Варіант 3 — система бронювання.



\## Завдання



Створити класи `Room`, `Reservation` і `Customer`.



Реалізувати generic-компонент для зв’язування клієнта з ресурсом і generic-метод для перевірки або вибору записів.



\## Опис проєкту



У цій лабораторній роботі реалізовано просту систему бронювання кімнат.



У програмі можна створювати клієнтів, кімнати та записи бронювання. Також реалізовано перевірку зайнятості кімнати, відбір підтверджених бронювань і вибір мінімального об’єкта за допомогою generic-методу.



\## Основні класи



`Customer` — клас, який описує клієнта системи бронювання.



`Room` — клас, який описує кімнату. Кімната має номер, тип і ціну.



`Reservation` — клас, який описує запис бронювання. Він містить клієнта, кімнату, дату початку, дату завершення та ознаку підтвердження.



`BookingLink<C, R>` — generic-клас для типобезпечного зв’язування клієнта з ресурсом.



`BookingUtils` — допоміжний клас із generic-методами для роботи з бронюваннями.



`Main` — демонстраційна програма.



\## Generic-клас



У роботі реалізовано generic-клас:



`BookingLink<C, R>`



Він використовується для зв’язування клієнта з різними типами ресурсів.



Приклади використання:



`BookingLink<Customer, Room>`



`BookingLink<Customer, Reservation>`



Тобто один і той самий generic-клас може працювати з різними типами даних без використання `Object` і без небезпечного приведення типів.



\## Generic-методи



У класі `BookingUtils` реалізовано generic-методи:



`getConfirmed(List<T>)`



Метод повертає список тільки підтверджених бронювань.



`isRoomBusy(List<T>, Room, LocalDate, LocalDate)`



Метод перевіряє, чи зайнята кімната на вказаний проміжок дат.



`findSmallest(List<T>)`



Метод знаходить мінімальний елемент у списку. Для кімнат це найдешевша кімната, а для бронювань — найперше бронювання за датою.



\## Обмеження типів



У роботі використано обмеження параметрів типу:



`T extends Reservation`



Це означає, що метод може працювати з об’єктами класу `Reservation` або його нащадками.



`T extends Comparable<? super T>`



Це означає, що метод може працювати тільки з тими об’єктами, які можна порівнювати між собою.

\## Запуск програми



Для запуску демонстраційної програми в IntelliJ IDEA потрібно відкрити файл `Main.java` і натиснути зелений трикутник біля методу `main`.

\## Висновок



У роботі було реалізовано систему бронювання з використанням generics. Generic-клас `BookingLink<C, R>` дозволяє типобезпечно зв’язувати клієнта з різними ресурсами. Generic-методи в `BookingUtils` дають змогу обробляти списки бронювань і порівнюваних об’єктів без дублювання коду.
