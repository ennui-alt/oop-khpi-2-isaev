\# Звіт до лабораторної роботи №14



\## Тема роботи



Базова багатопотоковість у Java.



\## Мета роботи



Метою лабораторної роботи є ознайомлення з базовими засобами створення та запуску потоків у Java, а також демонстрація виконання кількох незалежних задач у межах одного консольного застосунку.



\## Індивідуальний варіант



Варіант №3 — інтернет-магазин.



Для цього варіанта було реалізовано предметну область, пов’язану з обробкою замовлень, перевіркою описів товарів та формуванням накладних.



\## Постановка задачі



Необхідно було створити Java-застосунок, у якому декілька незалежних задач виконуються у власних потоках.



У роботі потрібно було показати:



\- створення потоку через `Thread`;

\- створення задачі через `Runnable`;

\- запуск потоків методом `start()`;

\- імітацію тривалої роботи через `Thread.sleep()`;

\- отримання імені поточного потоку;

\- очікування завершення потоків через `join()`;

\- тестування логіки, яка не залежить від порядку виконання потоків.



\## Опис предметної області



У програмі розглядається спрощена модель інтернет-магазину.



Основними сутностями є:



\- `Product` — товар з назвою, описом та ціною;

\- `Order` — замовлення покупця зі списком товарів;

\- `Invoice` — накладна, яка створюється для замовлення.



Окремо реалізовано клас `StoreService`, який містить основну логіку роботи з товарами, замовленнями та накладними.



\## Реалізація багатопотоковості



У програмі реалізовано три незалежні задачі.



`OrderProcessingThread` наслідує клас `Thread` і відповідає за обробку замовлень. Цей клас демонструє перший спосіб створення потоку в Java.



`ProductDescriptionTask` реалізує інтерфейс `Runnable` і виконує перевірку описів товарів. Якщо опис товару занадто короткий, товар додається до окремого списку.



`InvoiceTask` також реалізує `Runnable` і формує накладні для створених замовлень.



Усі задачі запускаються з головного класу `Main`. Після запуску потоків головна програма очікує їх завершення за допомогою `join()`.



\## Особливості виконання потоків



Проміжні повідомлення в консолі можуть з’являтися у різному порядку. Це нормально для багатопотокової програми, оскільки порядок виконання потоків визначається планувальником потоків Java.



Правильність програми не залежить від того, який потік завершиться першим. Фінальне повідомлення виводиться лише після завершення всіх робочих потоків.



\## Структура проєкту



Основний код розділено на кілька пакетів:



```text

demo — демонстраційний клас Main

model — класи предметної області

service — сервісна логіка програми

threads — класи потоків і Runnable-задач

```



Тести розміщено окремо у пакетах:



```text

demo

service

threads

```



\## UML-діаграми



Для роботи підготовлено UML-діаграму класів та діаграму сценарію запуску потоків.



Діаграма класів показує зв’язки між моделями, сервісом і класами потоків.



Діаграма сценарію демонструє запуск потоків через `start()` та очікування їх завершення через `join()`.



\## Тестування



Для перевірки роботи програми використано JUnit 5.



У тестах перевіряється:



\- запуск головної демонстраційної програми;

\- робота сервісного класу `StoreService`;

\- обробка замовлень;

\- перевірка коротких описів товарів;

\- формування накладних.



Точний порядок виконання потоків у тестах не перевіряється, оскільки він може змінюватися при різних запусках програми.



\## Запуск програми



Головний клас для запуску:



```text

ua.khpi.oop.lab14.demo.Main

```



\## Запуск тестів



Для запуску тестів використовується команда:



```bash

```



\## Результат виконання



У результаті виконання програми в консолі відображаються повідомлення від різних потоків.



Програма показує:



\- оброблені замовлення;

\- товари з коротким описом;

\- сформовані накладні;

\- повідомлення про завершення всіх потоків.



\## Висновок



Під час виконання лабораторної роботи було створено консольний Java-застосунок для демонстрації базової багатопотоковості. У програмі реалізовано створення потоків двома способами: через `Thread` та через `Runnable`. Було використано методи `start()`, `sleep()` і `join()`, а також показано, що порядок виконання потоків не є фіксованим. Робота дозволила закріпити навички організації простих незалежних задач у багатопотоковій Java-програмі.
