\# REPORT.md



\# Лабораторна робота №16



\## Тема роботи



Автоматизована збірка, тестування і CI для Java-проєкту.



\## Мета роботи



Мета лабораторної роботи — навчитися готувати Java-проєкт до автоматизованої збірки, запуску тестів і перевірки через GitHub Actions, а також реалізувати взаємодію Java-коду з native-кодом за допомогою JNI.



\## Індивідуальний варіант



Номер варіанта визначається за формулою:



```text

V = ((N - 1) mod 15) + 1

```



Для `N = 3`:



```text

V = ((3 - 1) mod 15) + 1 = 3

```



Отже, індивідуальний варіант — \*\*№3: середнє значення\*\*.



Завдання полягає в тому, щоб обчислити середнє арифметичне значення елементів масиву цілих чисел.



\## Короткий опис реалізованої операції



У програмі обчислюється середнє арифметичне значення елементів масиву.



Наприклад, для масиву:



```text

\[10, 20, 30, 40, 50]

```



сума елементів дорівнює:



```text

10 + 20 + 30 + 40 + 50 = 150

```



Кількість елементів:



```text

5

```



Середнє значення:



```text

150 / 5 = 30.0

```



У роботі ця операція реалізована двома способами:



\- засобами Java;

\- мовою C++ через JNI.



\## Структура каталогів проєкту



Проєкт має стандартну структуру Java-проєкту з додаванням каталогів для native-коду, JNI-заголовків та GitHub Actions.

Для проєкту використано \*\*Maven\*\*.



Файл конфігурації:

pom.xml

Maven використовується для компіляції Java-коду та запуску автоматичних тестів.



\## Java-реалізація



Java-реалізація знаходиться у класі:



```text

AverageCalculator

```



Основний метод:



```java

public double average(int\[] values)

```



Метод перевіряє вхідний масив, обчислює суму його елементів і повертає середнє арифметичне значення.



Основна логіка:



```java

long sum = 0;



for (int value : values) {

&#x20;   sum += value;

}



return (double) sum / values.length;

```



Також передбачено перевірку неправильних вхідних даних:



\- масив не може бути `null`;

\- масив не може бути порожнім.



\## Native-реалізація



Native-реалізація знаходиться у файлі:



```text

native/nativeaverage.cpp

```



Вона реалізує ту саму операцію, що й Java-код, але мовою C++.



Функція має JNI-сигнатуру:



```cpp

JNIEXPORT jdouble JNICALL Java\_ua\_khpi\_oop\_lab16\_service\_NativeAverageCalculator\_average

&#x20;       (JNIEnv \*env, jobject object, jintArray values)

```



У C++ коді масив отримується через JNI:



```cpp

jint \*elements = env->GetIntArrayElements(values, nullptr);

```



Після обчислення суми пам’ять звільняється:



```cpp

env->ReleaseIntArrayElements(values, elements, JNI\_ABORT);

```



Результат повертається як `jdouble`.



\## Java-клас із native-методом



Для виклику C++ реалізації створено клас:



```text

NativeAverageCalculator

```



У ньому native-бібліотека завантажується за допомогою:



```java

System.loadLibrary("nativeaverage");

```



Native-метод оголошено так:



```java

public native double average(int\[] values);

```



Під час запуску JVM шукає бібліотеку `nativeaverage` у каталозі, який вказано через `java.library.path`.



\## Команда генерації JNI header-файла



JNI header-файл було згенеровано командою:



```powershell

javac -encoding UTF-8 -h native-headers -d target/classes src/main/java/ua/khpi/oop/lab16/service/NativeAverageCalculator.java

```



У результаті було створено файл:



```text

native-headers/ua\_khpi\_oop\_lab16\_service\_NativeAverageCalculator.h

```



Цей файл підключається у C++ коді:



```cpp

\#include "ua\_khpi\_oop\_lab16\_service\_NativeAverageCalculator.h"

```



\## Команда збирання native-бібліотеки



На Windows native-бібліотека була зібрана командою:



```powershell

clang++ -std=c++17 -shared `

\-I"$env:JAVA\_HOME\\include" `

\-I"$env:JAVA\_HOME\\include\\win32" `

\-Inative-headers `

native\\nativeaverage.cpp `

\-o build\\native\\nativeaverage.dll

```



Після виконання команди було створено файл:



```text

build/native/nativeaverage.dll

```



Цей файл використовується Java-програмою під час виклику native-методу.



\## Завантаження native-бібліотеки



Для запуску програми необхідно вказати шлях до папки, у якій знаходиться native-бібліотека.



У IntelliJ IDEA для цього було додано VM options:



```text

\-Djava.library.path=D:\\IntelRaboti2\\oop-khpi-2-isaev\\lab16\\build\\native --enable-native-access=ALL-UNNAMED

```



Параметр `java.library.path` дозволяє JVM знайти файл `nativeaverage.dll`.



Параметр `--enable-native-access=ALL-UNNAMED` використовується для усунення попередження Java 25 про native-доступ.

\## GitHub Actions workflow



Для автоматичної перевірки створено файл:



```text

.github/workflows/ci.yml

```



Workflow запускається під час:



\- `push`;

\- `pull\_request`.



Основні кроки workflow:



1\. отримання коду репозиторію;

2\. встановлення JDK;

3\. генерація JNI header-файла;

4\. збирання native-бібліотеки;

5\. компіляція Java-проєкту;

6\. запуск JUnit-тестів.



На Linux runner бібліотека збирається як:



```text

build/native/libnativeaverage.so

```



Це відповідає механізму `System.loadLibrary("nativeaverage")`, який на Linux шукає бібліотеку з префіксом `lib` і розширенням `.so`.

\## Технічна проблема та спосіб її усунення



Під час виконання лабораторної роботи виникла проблема зі збиранням native-бібліотеки.



Спочатку команда:



```powershell

clang++ --version

```



не виконувалася, оскільки LLVM не був доступний у змінній середовища `PATH`.



Проблему було вирішено встановленням LLVM та додаванням шляху до компілятора:



```powershell

$env:Path="C:\\Program Files\\LLVM\\bin;$env:Path"

```



Після цього `clang++` почав працювати.



Також під час збирання виникла помилка:



```text

fatal error: 'stdio.h' file not found

```



Причина полягала у відсутності або неповному встановленні C++ компонентів Visual Studio Build Tools.



Проблему було вирішено через Visual Studio Installer, де було додано компонент:



```text

Desktop development with C++

```



Після цього native-бібліотеку вдалося зібрати у файл:



```text

nativeaverage.dll

```
